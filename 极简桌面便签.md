# 一、最终产品定位

## 1. 产品一句话

**一个可以贴在屏幕顶部的极简 Markdown 便签。**

它不是 Obsidian，不是 Notion，不是完整笔记软件，而是：

```txt
临时记录 + 任务规划 + 屏幕置顶提醒 + Markdown 导出
```

## 2. 核心使用场景

你真正需要的是：

```txt
写代码时，把今日任务贴在屏幕右上角
写论文时，把临时想法贴在屏幕边缘
开会时，快速记几条 Markdown
复习/做项目时，把计划贴在最上层
```

所以它应该长这样：

```txt
┌──────────────────────────────┐
│ 今日计划              📌 ⬇ ⚙ │
├──────────────────────────────┤
│                              │
│  # 今日计划                  │
│                              │
│  - [ ] 改论文                │
│  - [ ] 跑实验                │
│  - [ ] 整理 PPT              │
│                              │
└──────────────────────────────┘
```

# 二、技术路线总览

## 推荐技术栈

```txt
桌面框架：Tauri 2
前端框架：Svelte
语言：TypeScript + Rust
样式：Tailwind CSS
Markdown 渲染：markdown-it
HTML 安全清洗：DOMPurify
本地存储：JSON 文件
系统功能：
  - global-shortcut
  - autostart
  - tray
  - fs
  - dialog
  - window API
```

## 为什么是这个组合？

|技术|作用|为什么选它|
|---|---|---|
|Tauri 2|桌面壳|比 Electron 更轻，适合小工具|
|Svelte|UI 框架|比 React 更适合这种简单小应用|
|TypeScript|前端逻辑|类型安全，后期维护方便|
|Tailwind|样式|快速做美观卡片、圆角、阴影|
|markdown-it|Markdown 渲染|轻量，够用|
|DOMPurify|清理 HTML|防止 Markdown 里嵌入危险 HTML|
|JSON 文件|本地存储|数据量小，不需要数据库|

Tauri 2 的窗口 API 支持 `setAlwaysOnTop`，适合实现置顶便签。([Tauri](https://v2.tauri.app/reference/javascript/api/namespacewindow/?utm_source=chatgpt.com "window"))  
Tauri 也支持自定义窗口、透明窗口、自定义标题栏等，这对做“漂亮便签卡片”很有用。([Tauri](https://v2.tauri.app/learn/window-customization/?utm_source=chatgpt.com "Window Customization"))

# 三、为什么不选其他技术栈？

## 1. 不选 Electron

Electron 能做，但对你的需求太重。

你只是要一个桌面便签，不需要内置完整 Chromium + Node.js。Electron 更适合 VS Code、Slack、Discord 这种复杂应用。

## 2. 不优先选 Flutter

Flutter 跨平台好，UI 也漂亮，但对你这个需求有点过度。

它适合完整应用，不适合只做一个极简桌面便签。

## 3. 不优先选 Rust + egui

Rust + egui 会更轻，但问题是：

```txt
美观成本高
Markdown 排版麻烦
系统功能要自己组合
开发效率不如 Tauri
```

如果你的目标是“极限性能”，可以选 egui。  
但你的目标是“轻量 + 美观 + 快速做出来”，所以 Tauri 更平衡。

## 4. 不选原生 Swift + WPF/WinUI

原生当然最轻，但你要写两套：

```txt
macOS：Swift / SwiftUI
Windows：WPF / WinUI
```

这对个人项目不划算。

# 五、交互设计路线

你说“不要做编辑器页面”，所以我建议做成 **原地编辑模式**。

## 1. 默认状态：Markdown 预览

平时显示渲染后的 Markdown：

```md
# 今日计划

- [ ] 改论文
- [ ] 跑实验
- [ ] 整理 PPT
```

## 2. 点击正文：进入编辑状态

点击内容区域后，原地变成 textarea：

```md
# 今日计划

- [ ] 改论文
- [ ] 跑实验
- [ ] 整理 PPT
```

## 3. 退出编辑

触发方式：

```txt
失焦
按 Esc
按 Ctrl/Cmd + Enter
```

退出后重新渲染 Markdown。

## 4. 顶部工具栏

只保留几个按钮：

```txt
📌 置顶
⬇ 导出
⚙ 设置
× 隐藏
```

不要放太多。

# 八、导出规则

导出成 Markdown 文件。

## 文件名规则

```txt
YYYYMMDD_HHmmss_标题.md
```

例如：

```txt
20260518_213522_今日计划.md
```

如果标题为空：

```txt
20260518_213522_untitled.md
```

## Markdown 内容格式

```md
# 今日计划

- [ ] 改论文
- [ ] 跑实验
- [ ] 整理 PPT
```

如果正文已经有一级标题，可以不重复插入标题。  
第一版可以简单处理：直接导出正文内容即可。
